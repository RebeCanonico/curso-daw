Animaciones con CSS

CSS permite animar elementos HTML sin necesidad de recurrir a JavaScript.

¿Que son las animaciones?

Una animacion es el cambio gradual de un estilo a otro.

Se pueden cambiar tantas propiedades de CSS como se quiera, tantas veces como sea necesario.

Para usar una animación debe especificarse la regla @keyframes.

La regla @keyframes

Esta regla contiene los estilos que el elemento tendrá en determinados momentos.

Cuando se especifica una regla los cambios se ejecutaran de manera gradual en ciertos momentos.

Para que la animacion funcione, debes asignar un keyframe al CSS de un elemento.

Propiedades de las animaciones

  • animation-delay
  • animation-iteration-count
  • animation-direction
  • animation-timing-function
  • animation-fill-mode

animation-delay

Especifica el retardo dekl inicio de una animación.

Los valores positivos indican el tiempo que ha de transcurrir para que la animacion de comienzo. Los valores negativos indican que la animacion empezará como si ya hubiese transcurrido el tiempo especificado.

animation-iteration-count

Establece el numero de veces que se repetirá una animación.

Se puede establecer un número natural de veces o se puede indicar un numero ilimitado con el valor infinite.

animation-direction

Especifica si la animación se producira en orden normal, inverso o irá alternando ciclos. Los valores posibles son:

  • normal la animación se reproduce hacia adelante. Valor por defecto.
  • reverse la animación se reproduce hacia atrás. (de 100% a 0%).
  • alternate la animación se reproduce hacia adelante, y luego hacia atrás.
  • alternate-reverse la animación se reproduce hacia atrás, y luego hacia adelante.

animacion-timing-function

Puede tener los siguientes valores:

  • ease La animacion tendrá un inicio lento, luego irá parido y tendra un final lento. Valor por defecto.
  • linearLa animación tendrá una velocidad estatica durante toda la duración.
  • ease-in La animación tendrá un inicio lento.
  • ease-outLa animación tendrá un final lento y un final lento.
  • ease-in-outLa animación tendrá un inicio lento.
  • cubic-bizier(n, n, n, n)Permite al usuario especificar la curva de velocidad mediante una función.

animation-fill-mode

Por defecto, las animacones no afectan a los elementos antes de iniciarse o despues de finalizar. Esta propiedad cambia este comportamiento.

La propiedad animation-fill-mode establece el estilo del elemento objetivo cuando la animación no se está reproduciendo (antes de empezar, despues de finalizar o ambas).

Los valores posibles:

  • none Valor por defecto. La animacion no aplicará cambios de estilo.
  • forwards El elemento retendrá el estilo establecido en el ultimo keyframe de la animacion (dependera si se está reproduciendo normal o de manera invertida animation-direction y del numero de iteraciones. animation-iteration-count).
  • backwardsEl elemento retendrá el estilo establecido en el primer keyframe d ela animación. Dependerá de la direccióny tomará el estilo durante el retardo proporcionado por animation-delay.
  • both La animación seguirá las reglas combinadas de forwards y backwards.
